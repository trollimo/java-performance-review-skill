# Adding New Performance Rules

Version 1.0

Этот документ описывает единый шаблон добавления новых правил.

---

## Алгоритм

1. Найти технологию.

Например

Spring

Kafka

PostgreSQL

Java

или создать новую папку.

---

2. Определить тему.

Например

spring/cache.md

spring/webclient.md

postgresql/indexes.md

---

3. Если файла нет —

создать его.

---

4. Каждое правило получает уникальный ID.

Пример

SPR-101

PG-203

JAVA-315

---

5. Использовать шаблон

# XXX-001

## Название

...

Severity

Critical

High

Low

...

...

---

...

...

...

---

---

---

## Требования

Каждое правило

✔ объясняет причину

✔ объясняет последствия

✔ предлагает исправление

✔ содержит Severity

✔ содержит Confidence

✔ содержит Expected Improvement

✔ имеет уникальный ID

---

## Не допускается

- дублировать существующие правила

- делать правило без объяснения

- использовать расплывчатые формулировки

- использовать слово "может" без пояснения

---

## Если найдено новое семейство ошибок

Создать новый markdown-файл.

Не увеличивать существующие файлы бесконечно.

Оптимальный размер файла

300–1200 строк.

---

## После добавления

Проверить

- Related Rules

- Critical List

- Checklist

в конце файла.
