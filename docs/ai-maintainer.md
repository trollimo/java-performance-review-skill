# AI Maintainer Guide

Этот документ предназначен для ИИ, который будет развивать данный репозиторий.

---

## Цель

Сделать репозиторий максимально полной базой знаний по поиску проблем производительности Java Enterprise систем.

Не изменять существующие правила без необходимости.

Предпочитать добавление новых.

---

## При каждой доработке

1.

Просканировать весь репозиторий.

---

2.

Найти

- пустые файлы

- почти пустые файлы

- незаполненные разделы

---

3.

Заполнить их.

---

4.

Проверить

нет ли технологий,

для которых отсутствуют правила.

Например

Redis

Oracle

MongoDB

RabbitMQ

Elasticsearch

gRPC

Netty

R2DBC

Reactor

Virtual Threads

Micrometer

Prometheus

---

5.

Если технология отсутствует —

создать новую папку.

---

6.

Для каждой новой технологии

создать минимум

20 правил.

---

7.

Использовать шаблон

из

docs/adding-new-rules.md

---

8.

Не изменять ID существующих правил.

---

9.

Не удалять правила.

Допускается только

Deprecated.

---

10.

Поддерживать размер файла

не более

1200 строк.

При превышении —

разделить файл.

---

## После каждой серии изменений

Проверить

- нет ли дублирования

- нет ли одинаковых правил

- есть ли Related Rules

- есть ли Critical List

---

## Главная цель

Каждая новая версия репозитория должна содержать больше практических знаний, чем предыдущая.
