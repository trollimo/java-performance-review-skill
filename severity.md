# Severity Model

Version: 1.0

---

# Цель

Severity определяет приоритет исправления проблемы.

Критичность оценивается независимо от вероятности возникновения.

Например

Редкий Deadlock

может иметь

Severity = Critical

Confidence = Low

---

# Severity Levels

| Severity | Priority | Исправление |
|-----------|----------|-------------|
| Critical | P0 | Немедленно |
| High | P1 | Ближайший релиз |
| Medium | P2 | Плановое исправление |
| Low | P3 | По возможности |
| Info | P4 | Рекомендация |

---

# CRITICAL

Использовать только если проблема способна привести к одной из следующих ситуаций.

## Production Outage

Например

- OutOfMemoryError
- Deadlock
- бесконечная рекурсия
- исчерпание connection pool
- исчерпание thread pool

# Confidence

Severity

не зависит

от Confidence.

Пример

Critical

Confidence = Low

означает

возможная очень серьезная проблема.

---

# Рекомендации по исправлению

Critical

исправлять немедленно

High

до следующего релиза

Medium

в плановом порядке

Low

по возможности

Info

не требует обязательного исправления

---

# Правило

При наличии сомнений

не понижать Severity.

Лучше снизить Confidence.

Пример

Severity = Critical

Confidence = Low

чем

Severity = Medium

если последствия потенциально катастрофические.
